Modding Guide

Three tiers of extensibility, from drag-and-drop arena maps to full Java API integration.

Overview

Craftics supports three levels of modding:

  1. Custom Arena Maps: Build .nbt/.schem structures in Minecraft, no code required.
  2. Datapack Biome Modding: JSON files for custom biomes, enemies, and loot.
  3. Fabric Mod Integration: Java API for custom AI and programmatic biomes.

Custom Arena Maps

Step 1: Build Your Arena

Create the full scene in creative mode including a flat area for the grid, surrounding decorations (trees, ruins, water, cliffs), and any blocks you want. The area outside the grid is purely cosmetic. The game only reads the grid boundaries and spawn markers.

Step 2: Place Marker Blocks

Place special marker blocks to define the playable grid and spawn positions. The game scans for these blocks when loading the arena.

┌─────────────────────────────┐ │ Decorations... │ │ EMERALD─────────────────EMERALD │ ← Emerald corners OUTSIDE the grid │ │ GOLD │ │ ← Gold = Player 1 spawn (inside grid) │ │ (playable grid) │ │ │ │ IRON │ │ ← Iron = Player 2 spawn (optional) │ │ │ │ │ EMERALD─────────────────EMERALD │ │ More terrain... │ └─────────────────────────────┘

Marker Block Reference

Block Purpose
Emerald Block Arena boundary corner. Place 2+ on opposite corners outside the grid.
Gold Block Player 1 spawn (primary)
Iron Block Player 2 spawn (multiplayer)
Copper Block Player 3 spawn
Coal Block Player 4 spawn

All markers must be at the same Y level.

Step 3: Export

Option A: WorldEdit (recommended)

  1. Select: //pos1 and //pos2
  2. Copy: //copy
  3. Save: //schem save <name>
  4. Find in config/worldedit/schematics/
  5. Copy to craftics_arenas/<biome>/<number>.schem

Option B: Vanilla Structure Block (48×48×48 limit)

  1. /give @s structure_block
  2. Set to Save mode, define box, name it, click SAVE
  3. Find in <world>/generated/minecraft/structures/
  4. Copy to datapack at data/craftics/structures/arenas/<biome>/<number>.nbt

Selection Rules

Files are scanned sequentially (1, 2, 3…) and the scanner stops at the first missing number. One map is picked randomly each fight. Boss maps (boss_1, boss_2…) are used for boss levels, falling back to regular maps if no boss maps exist.

Tips

  • Size your grid for the biome's growth range (e.g., Plains starts 8×8, grows +1 per level).
  • The isometric camera looks from the southwest at ~55°. Tall structures on the north/east edges may block the view.
  • Floor level = marker Y level.
  • Obstacles are overlaid by the game, so your arena provides the base terrain only.
  • Add atmosphere outside the grid (lanterns, campfires, particle-emitting blocks, etc.).
  • Under-floor lighting is automatic (glowstone every 3 blocks).

Custom Biomes

JSON Schema

Each custom biome is defined by a single JSON file. Here is the full schema with all supported fields: 

{
  "id": "my_biome",
  "name": "My Custom Biome",
  "order": 20,
  "levels": 5,
  "grid": {
    "base_width": 10,
    "base_height": 10,
    "width_growth": 1,
    "height_growth": 1
  },
  "floor_blocks": ["minecraft:stone_bricks", "minecraft:mossy_stone_bricks"],
  "obstacle_blocks": ["minecraft:cobblestone_wall"],
  "obstacle_density": 0.1,
  "obstacle_density_growth": 0.02,
  "environment": "cave",
  "night": true,
  "enemies": {
    "passive": [
      {"type": "minecraft:bat", "weight": 3, "hp": 2, "attack": 0, "defense": 0, "range": 1}
    ],
    "hostile": [
      {"type": "minecraft:skeleton", "weight": 8, "hp": 10, "attack": 3, "defense": 0, "range": 3},
      {"type": "minecraft:spider", "weight": 6, "hp": 8, "attack": 3, "defense": 0, "range": 1}
    ],
    "boss": {"type": "minecraft:warden", "hp": 50, "attack": 8, "defense": 4, "range": 2}
  },
  "loot": [
    {"item": "minecraft:diamond", "weight": 5},
    {"item": "minecraft:iron_ingot", "weight": 15}
  ]
}

Field Reference

Field Type Description
id string Required. Unique identifier.
name string Display name in HUD and level select.
order int Sort position. Lower = earlier.
levels int Levels in this biome (typically 5). Last = boss.
grid.base_width int Starting grid width.
grid.base_height int Starting grid height.
grid.width_growth int Width increase per level.
grid.height_growth int Height increase per level.
floor_blocks string[] Floor tile blocks (checkerboard pattern).
obstacle_blocks string[] Obstacle blocks.
obstacle_density float Base obstacle probability (0.0–1.0).
obstacle_density_growth float Density increase per level.
environment string Visual style (see list below).
night bool Night mode (prevents undead burning).
enemies.passive array Passive mob pool.
enemies.hostile array Hostile mob pool (weighted random selection).
enemies.boss object Boss for the final level.
loot array Weighted loot table.

Environment Styles

plains, forest, desert, jungle, river, mountain, snowy, cave, nether_wastes, end, deep_dark

Creating, Overriding, Combining

  • Place your biome JSON at data/craftics/craftics/biomes/my_biome.json
  • Use the same id as a built-in biome to override its settings.
  • Multiple JSON files with unique IDs create multiple biome variations.
  • Combine custom biome JSONs with multiple .nbt maps per biome for maximum variety.

Enemy Definitions

Each enemy entry in the passive or hostile arrays follows this format: 

{"type": "minecraft:zombie", "weight": 8, "hp": 6, "attack": 2, "defense": 0, "range": 1}
Field Type Description
type string Minecraft entity ID. Must be a supported mob type.
weight int Spawn weight. Higher = more frequent.
hp int Hit points for this enemy.
attack int Attack power.
defense int Defense value.
range int Attack range in tiles.

Supported Mob Types

All mobs below have registered AI classes and are supported in custom biomes:

  • Passive: minecraft:cow, minecraft:pig, minecraft:sheep, minecraft:chicken, minecraft:rabbit, minecraft:bat, minecraft:cod, minecraft:salmon, minecraft:axolotl, minecraft:cat, minecraft:ocelot, minecraft:bee, minecraft:wolf, minecraft:goat, minecraft:polar_bear, minecraft:llama
  • Hostile: minecraft:zombie, minecraft:husk, minecraft:drowned, minecraft:skeleton, minecraft:stray, minecraft:bogged, minecraft:spider, minecraft:cave_spider, minecraft:creeper, minecraft:vindicator, minecraft:pillager, minecraft:witch, minecraft:enderman, minecraft:phantom, minecraft:silverfish, minecraft:slime, minecraft:blaze, minecraft:ghast, minecraft:shulker, minecraft:breeze, minecraft:magma_cube, minecraft:wither_skeleton
  • Boss: minecraft:warden, minecraft:ender_dragon

Loot Tables

Each loot entry specifies an item and a weight for random selection: 

{"item": "minecraft:diamond", "weight": 5}

1–3 items are rolled per level completion. Higher weight means that item is chosen more frequently relative to other entries in the table.

Datapack Setup

Wrap your custom biomes and arena structures into a standard Minecraft datapack: 

my_arena_pack/
  pack.mcmeta
  data/
    craftics/
      craftics/
        biomes/
          my_custom_biome.json
      structures/
        arenas/
          my_custom_biome/
            1.nbt
            2.nbt
            boss_1.nbt

pack.mcmeta

{
  "pack": {
    "pack_format": 48,
    "description": "Custom Craftics arenas"
  }
}

Place the finished datapack folder in your world’s datapacks/ directory.

Hot Reload

  • Biome JSONs: Run /reload to reload without restarting the server.
  • Structure files: Loaded on-demand when combat starts, so new files are picked up immediately.
  • No server restart needed for any modding content.

Java API

For full programmatic control, Craftics exposes a Fabric mod API. Register custom AI behaviors, biomes, and more from Java code.

Custom AI Registration

import com.crackedgames.craftics.combat.ai.AIRegistry;

AIRegistry.register("mymod:custom_mob", (self, arena, playerPos) -> {
    // Your AI logic here
    return new EnemyAction.Attack(self.getAttackPower());
});

Biome & Level API

CrafticsAPI.registerBiome(myBiomeTemplate);
int total = CrafticsAPI.getTotalLevels();

Action Types

13 action types are available for custom AI:

Move, Attack, MoveAndAttack, Flee, Teleport, TeleportAndAttack, Pounce, Explode, RangedAttack, Swoop, StartFuse, Detonate, Idle

Addon Policy

Allowed

  • Addons via API (biomes, enemies, AI, items)
  • Datapacks (arenas, loot, biome configs)
  • Resource packs retexturing Craftics content
  • Modpacks on CurseForge/Modrinth with credit
  • Videos, streams, reviews (no permission needed)

Not Allowed

  • Redistributing Craftics outside modpack launchers
  • Copying source code into your projects
  • Re-uploading without permission
